Overview
The Online Food Ordering System is a Database Management System project that efficiently manages restaurants, customers, menu items, and orders. It demonstrates database design, normalization, and implementation using SQL in a real-world scenario.

Aim
To design and implement a relational database for an online food ordering platform that ensures data accuracy, integrity, and quick information retrieval.

Objectives
Identify and design entities and relationships.

Normalize the database up to Third Normal Form (3NF).

Perform DDL and DML operations.

Retrieve data using queries and joins.

Apply constraints to maintain data consistency.

Tools & Technologies
Database: MySQL / Oracle / PostgreSQL

Frontend (optional): HTML, PHP / Flask

Platform: MySQL Workbench / XAMPP / phpMyAdmin

Documentation: MS Word / Google Docs

System Requirements
Hardware: Intel i3 or higher, minimum 4GB RAM, 10GB free space
Software: Windows or Linux, MySQL or XAMPP, MySQL Workbench

Database Design
Entities: Restaurant, Menu_Item, Customer, Orders, Order_Details

Relationships:
One restaurant offers many menu items

One customer can place many orders

Each order contains multiple menu items

Data is stored in relational tables using primary and foreign keys to maintain integrity.
